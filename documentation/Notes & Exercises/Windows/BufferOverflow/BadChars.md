### Comparing Badchars with Mona Scripts

You can use Mona scripts to search for a byte-array.   The commands below will generate a bytearray.bin and bytearray.txt file.  These can also be used to grab the character set for your BoF code as well.

#### Full Byte-Array

If you are working with the full set of characters 00-FF:

```bash
# Set the location of the char file
!mona config -set workingfolder c:\somefolder\
# Generate the char file
!mona bytearray 	# You can copy this bytearray from the bin or txt file to your exploit code
# Run your exploit, then compare the memory with your bytearray file, specifying the address where it should be
!mona compare -f c:\somefolder\bytearray.bin -a 03aef764
# If it reports a bad character, remove that from the bytearray with this command, specifying the character
!mona bytearray -cpb "\x00"		# Update your script by removing this character or copy/paste the bytearray file to it
# Re-run the exploit and compare again, removing any other bad characters.
```

#### Partial Bytes

If you have limited space in your buffer and can only compare subsets of the full character set at a time:

```bash
# Set the location of the char file
!mona config -set workingfolder c:\somefolder\
# Have mona create a partial binary file.  You are basically specifying an exclusion range, similar to how you identified bad characters above.  It will generatea bin file containing the bits not included in the range you specify.
!mona bytearray -cpb "\x00\x20..\xff" #You can import the bin or txt file into your code
# Run the exploit, then compare the memory with your bytearray file, specifying the address where it appears
!mona compare -f c:\somefolder\bytearray.bin -a 03aef764
# Repeat this process, generating the next chunk of hex characters, re-running the code, the re-comparing.  This is tedious, but it works.
!mona bytearray -cpb "\x00..\x1f\x40..\xff"
!mona compare -f c:\somefolder\bytearray.bin -a 03aef764
# etc
```
